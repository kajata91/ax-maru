작업 후에는 아래 **작업 규칙**대로 README·memory.md를 갱신한다.

## 1. 정체성 · 목표

AX-Maru = AI·AX(AI 전환) 분야의 **최신 정보를 매일 큐레이션**하는 전문 리서치 에이전트 + 정보 허브.
원칙: **품질 > 속도** (많이가 아니라 잘 고른다). Obsidian으로 자료 관리, `site/`에 웹 대시보드, GitHub로 버전 관리.

## 2. 운영 방식 (단계 로드맵)

- **1단계(MVP, 현재):** Claude Desktop **Cowork**으로 매일 `_지시문.md`를 실행 → 그날 다이제스트를 `Daily/`에 생성·검증. 컴퓨터를 켜둔 채 수동/예약 실행.
- **2단계(자동화):** Claude Code 스크립트화 → 무인 상시 실행 + 08시 대시보드 + 이메일 발송 + 원문/번역 토글.

2단계 전환 신호: 밤사이 자동 수집 필요 / 08시 대시보드·이메일 자동화 필요 / 웹 UI·원문 토글 필요.

## 3. 일일 작업 절차 (Cowork 지시문 요지)

`수집 → 선별 → 가공 → 형식 → 저장 → 보고` 6단계.
1. **수집**: Tier-1 소스 중심으로 최근 24~72h 발행 소식·연구를 폭넓게 수집(후보 풀).
2. **선별**: 중요도·신뢰도·시의성 기준 **하루 총 20건 이내** 엄선. 같은 사건은 1건으로 묶고 출처 복수 표기.
3. **가공**: 한국어로 `요약 + HR·조직 관점 시사점`을 합쳐 **1,000자 이내**. 중요도(상/중/하) 부여, 출처·발행일·원문 링크 필수.
4. **형식**: 아래 카드 형식. 문서 상단에 (a)날짜 (b)총 건수 (c)이번 주 키워드 Top 5.
5. **원칙**: 날조·추측 금지, 미확인은 '미확인' 표기, 출처 불확실 항목은 제외. 영어 원문 근거 → 한국어 요약.
6. **보고**: 저장 후 파일 경로·총 건수·가장 중요한 3건을 한국어로 보고.

## 9. 작업 규칙 (중요)

1. **핵심 기능 변화는 README.md "변경 이력"에 한 줄 기록** (기능 추가·명칭 변경·동작 변경 등).
2. **모든 진행상황은 memory.md에 `## YYYY-MM-DD`로 시간순 누적** (무엇·왜·어떻게).
3. **커밋·푸시는 `/ax-push ["메시지"]` 스킬로** (변경분 자동 커밋 후 GitHub main 푸시).
4. 새 다이제스트를 만들면 `site/data/`의 `index.json`·해당 일자 JSON도 함께 갱신해 프론트와 동기화.
