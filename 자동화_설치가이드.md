# AX-Maru 자동화 설치 가이드 (초보자용 · 따라하기)

> 이 자동화는 **돈이 들지 않습니다.** (GitHub Actions·Pages·이메일 모두 무료)
> 자료 수집만 회원님이 Cowork로 직접 하고, 나머지(검증·웹공개·이메일)는 GitHub가 자동으로 합니다.
> **Mac을 꺼놔도** 공개·이메일은 GitHub 클라우드에서 처리됩니다.

---

## 0. 전체 그림 (한눈에)

```
회원님: Cowork로 그날 다이제스트 만들기  →  /ax-push (한 줄)
                                              │
                                              ▼  (GitHub가 자동으로)
                         ① 데이터 검증(링크·요약·중복 점검)
                         ② 웹사이트(GitHub Pages) 자동 갱신·공개
                         ③ 요약+링크 이메일 발송
```

회원님이 매일 하는 건 **① 다이제스트 만들기 → ② `/ax-push`** 두 가지뿐입니다.

---

## 1. (회원님이 1번만) GitHub Pages 켜기 — 웹사이트 공개

웹사이트 주소를 만드는 단계입니다.

1. 인터넷에서 👉 **https://github.com/kajata91/ax-maru/settings/pages** 접속 (로그인 상태)
2. **Build and deployment** 항목의 **Source**를 **`GitHub Actions`** 로 선택
3. 끝. (저장 버튼이 따로 없으면 선택만 하면 됩니다)

> 무료 공개를 위해 저장소가 **공개(public)** 여야 합니다. 비공개면
> 👉 https://github.com/kajata91/ax-maru/settings → 맨 아래 **Change repository visibility** → Public.

설정 후 잠시 뒤, 웹사이트 주소는 보통 **https://kajata91.github.io/ax-maru/** 가 됩니다.

---

## 2. (회원님이 1번만) 이메일 발송 준비 — Gmail 앱 비밀번호

이메일을 자동 발송하려면 Gmail "앱 비밀번호" 하나가 필요합니다. (일반 비밀번호 아님, 16자리 전용 코드)

**2-1. 앱 비밀번호 만들기**
1. 👉 https://myaccount.google.com/apppasswords 접속 (kajata91@gmail.com 로그인)
   - 이 화면이 안 열리면 먼저 **2단계 인증**을 켜야 합니다: https://myaccount.google.com/signinoptions/two-step-verification
2. 앱 이름에 `AX-Maru` 입력 → **만들기**
3. 화면에 뜨는 **16자리 코드**(예: `abcd efgh ijkl mnop`)를 복사 (공백은 빼도 됩니다)

**2-2. GitHub에 비밀로 저장 (3개)**
1. 👉 https://github.com/kajata91/ax-maru/settings/secrets/actions 접속
2. **New repository secret** 버튼으로 아래 3개를 하나씩 추가:

| Name (이름) | Secret (값) |
|---|---|
| `MAIL_USERNAME` | `kajata91@gmail.com` |
| `MAIL_PASSWORD` | 위 16자리 앱 비밀번호 |
| `MAIL_TO` | `kajata91@gmail.com` (받을 주소) |

> 이메일이 필요 없으면 이 2단계는 건너뛰어도 됩니다. (그러면 발송만 생략되고 나머지는 정상 작동)

---

## 3. 첫 실행 확인

1. 위 1·2번을 마친 뒤 👉 https://github.com/kajata91/ax-maru/actions 접속
2. **AX-Maru 검증·공개·이메일** 워크플로 → **Run workflow** 버튼 클릭
3. 초록색 ✓ 가 뜨면 성공. 웹사이트 주소가 열리고 이메일이 도착합니다.

> ⚠️ Pages를 켜기 전(1번 전)에 실행하면 빨간 ✗ 가 날 수 있는데 정상입니다. 1·2번을 마치고 다시 **Run workflow** 하면 됩니다.

---

## 4. 매일 사용법 (이게 일상)

1. **Cowork로 그날 다이제스트 만들기** — `AX-Maru_MVP_실행가이드.md`의 지시문 실행
   → `Daily/AX-Maru_Daily_YYYY-MM-DD.md` 와 `site/data/YYYY-MM-DD.json`, `index.json`이 갱신됩니다.
2. 저에게 **`/ax-push`** 라고 하기 (또는 "올려줘")
   → GitHub가 자동으로 검증 → 웹 갱신 → 이메일 발송.

끝입니다. Mac을 꺼도 ②③은 클라우드에서 처리됩니다.

---

## 5. 자동으로 점검되는 것 (무료 품질 가드)

`/ax-push` 하면 GitHub가 매번 자동 점검합니다:

- **JSON 무결성**: 항목에 제목·출처·링크·요약이 빠지면 **배포를 막고** 알려줌 (깨진 데이터 방지)
- **링크 점검**: 각 원문 링크가 실제로 열리는지 확인 (안 열리면 경고)
- **날짜 간 중복**: 같은 사건이 여러 날 반복되면 경고로 표시
- **6개 분야 커버리지**: 비어있는 분야가 있으면 알려줌

> 참고: 요약이 원문 내용과 **의미상** 맞는지까지의 심층 점검은 AI(유료)가 필요해서 지금은 하지 않습니다.
> 필요해지면 그때 비용을 안내드리고 허락을 받은 뒤 추가합니다.

---

## 6. 문제가 생기면

- 빨간 ✗: https://github.com/kajata91/ax-maru/actions 에서 해당 실행을 눌러 로그 확인 → 저에게 캡처/내용을 보여주시면 고쳐드립니다.
- 이메일이 안 옴: 2번 시크릿 3개가 정확한지, 앱 비밀번호가 맞는지 확인.
- 웹사이트가 안 열림: 1번 Pages 설정과 저장소 공개 여부 확인.
